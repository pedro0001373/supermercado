1. AUTENTICACAO E SEGURANCA (src/routes/auth.js)
================================================================================

POST /api/auth/login
  - Login com verificacao bcrypt
  - Registra log de sucesso/falha
  - Atualiza ultimo_acesso do usuario
  - Retorna: { usuario: { id, nome, login, perfil } }

GET /api/auth/usuarios
  - Lista todos os usuarios (id, nome, login, perfil, ativo, ultimo_acesso)
  - Nao retorna campo senha

POST /api/auth/usuarios
  - Cria novo usuario com senha bcrypt (minimo 4 caracteres)
  - Campos: nome*, login*, senha*, perfil (operador/gerente/admin)
  - Registra log: criar_usuario

PUT /api/auth/usuarios/:id
  - Edita usuario (nome, login, perfil, ativo)
  - Senha opcional (se enviada, cria novo hash bcrypt)
  - Registra log: editar_usuario

POST /api/auth/usuarios/:id/reset-senha
  - Reset de senha por admin (nova_senha minimo 4 chars)
  - Registra log: reset_senha

POST /api/auth/alterar-senha
  - Alterar propria senha (requer senha_atual correta)
  - Registra log: alterar_senha ou alterar_senha_falhou

GET /api/auth/logs
  - Lista logs de auditoria com filtro por modulo/usuario
  - Params: modulo, usuario_id, limit (default 100)

GET /api/auth/backups
  - Lista backups do sistema (arquivo, tamanho, data)

POST /api/auth/backups
  - Executa backup manual do banco de dados
  - Registra log: backup_manual

PERFIS DE ACESSO:
  operador: apenas PDV
  gerente:  dashboard, PDV, produtos, categorias, fornecedores, clientes,
            estoque, validade, notas-entrada, nfce, vendas, relatorios
  admin:    tudo (+ configuracoes, usuarios, logs, backups)

2. PRODUTOS (src/routes/produtos.js)
================================================================================

GET /api/produtos
  - Lista produtos com paginacao
  - Params: busca (nome ou codigo_barras), categoria_id, ativo, page, limit
  - Retorna: { produtos, total, page, pages }

  - Busca produto por codigo de barras (para PDV)
  - Retorna 404 se nao encontrado

GET /api/produtos/:id
  - Detalhes de um produto com nome da categoria

POST /api/produtos
  - Cria produto com todos os campos fiscais
  - Campos obrigatorios: nome, preco_venda
  - Campos opcionais: codigo_barras, descricao, categoria_id, unidade (UN/KG/LT/CX/PCT),
    preco_custo, margem_lucro, estoque_atual, estoque_minimo, ncm, cst, cfop,
    icms_aliquota, pis_aliquota, cofins_aliquota, peso_liquido, usa_balanca
  - Se estoque_atual > 0, cria movimentacao "Estoque inicial"

PUT /api/produtos/:id
  - Atualiza todos os campos do produto (exceto estoque_atual)

DELETE /api/produtos/:id
  - Soft delete (ativo = 0)

6. CONTROLE DE ESTOQUE (src/routes/estoque.js)
================================================================================

GET /api/estoque/alertas
  - Produtos com estoque_atual <= estoque_minimo

GET /api/estoque/movimentacoes
  - Historico de movimentacoes com filtros
  - Params: produto_id, tipo, data_inicio, data_fim, page, limit
  - Tipos: entrada, saida, inventario, perda

POST /api/estoque/entrada
  - Entrada manual de estoque (produto_id*, quantidade*, motivo, usuario)
  - Registra movimentacao e log

POST /api/estoque/saida
  - Saida manual de estoque (mesmo formato da entrada)

POST /api/estoque/inventario
  - Ajuste de inventario em lote
  - Body: { itens: [{ produto_id, quantidade_contada }], usuario }

POST /api/estoque/perda
  - Registra perda de produto (produto_id*, quantidade*, motivo, usuario)

--- PEDIDO DE COMPRA AUTOMATICO ---

GET /api/estoque/sugestao-pedido
  - Analisa consumo dos ultimos 30 dias
  - Calcula consumo diario, dias de estoque restantes
  - Sugere quantidade para 30 dias de estoque
  - Retorna: lista com consumo_diario, dias_estoque, quantidade_sugerida, custo_estimado

POST /api/estoque/pedido-compra
  - Cria pedido de compra (fornecedor_id, itens, observacoes, usuario)
  - Itens: [{ produto_id, quantidade_sugerida, quantidade, custo_estimado }]

GET /api/estoque/pedidos-compra
  - Lista ultimos 50 pedidos com qtd_itens

GET /api/estoque/pedido-compra/:id
  - Detalhes do pedido com itens (produto_nome, codigo_barras, unidade)

--- CODIGO DE BARRAS ---

POST /api/estoque/gerar-codigo-barras
  - Gera EAN-13 interno para 1 produto (produto_id)
  - Formato: prefixo(3) + sequencial(9) + digito verificador(1)
  - Prefixo configuravel (padrao: 789)

POST /api/estoque/gerar-codigo-barras-lote
  - Gera codigos para todos os produtos sem codigo de barras

GET /api/estoque/sem-codigo-barras
  - Lista produtos ativos sem codigo de barras

GET /api/estoque/etiqueta/:id
  - Dados do produto para impressao de etiqueta

--- CONTROLE DE LOTES ---

GET /api/estoque/lotes/:produtoId
  - Lista lotes de um produto com status de validade (ok/proximo/vencido)

POST /api/estoque/lotes
  - Cadastra lote manual (produto_id*, data_validade*, numero_lote, quantidade, etc)

GET /api/estoque/lote-rastreio/:loteId
  - Rastreabilidade completa: dados do lote + fornecedor + nota + vendas

POST /api/estoque/enviar-alerta
  - Dispara alerta manual de estoque baixo (email/WhatsApp)

--- ALERTAS AUTOMATICOS ---
  - Verificacao automatica a cada 1 hora
  - Envia email via SMTP (nodemailer) e/ou WhatsApp via webhook
  - Intervalo configuravel entre alertas (padrao: 24h)

9. PDV - FRENTE DE CAIXA (src/routes/caixa.js + vendas.js)
================================================================================

--- CAIXA ---

GET /api/caixa/aberto
  - Retorna caixa aberto atual com totais calculados
  - Inclui: total_vendas, total_sangrias, total_suprimentos, pagamentos_por_tipo

POST /api/caixa/abrir
  - Abre caixa (operador*, numero_caixa, valor_abertura)
  - Erro se ja houver caixa aberto

POST /api/caixa/fechar
  - Fecha caixa e calcula resumo completo
  - Calcula: esperado em caixa, diferenca
  - Body: valor_fechamento (contado), observacoes

POST /api/caixa/sangria
  - Registra retirada de dinheiro do caixa (valor*, motivo)

POST /api/caixa/suprimento
  - Registra entrada de dinheiro no caixa (valor*, motivo)

GET /api/caixa/historico
  - Ultimos 30 caixas com todos os dados

GET /api/caixa/:id/movimentacoes
  - Sangrias e suprimentos de um caixa especifico

--- VENDAS ---

GET /api/vendas
  - Lista vendas com filtros (data_inicio, data_fim, page, limit)

GET /api/vendas/:id
  - Detalhes da venda com itens e pagamentos

POST /api/vendas
  - Finaliza venda completa
  - Body: { itens, pagamentos, cliente_cpf, cliente_id, desconto }
  - Itens: [{ produto_id, codigo_barras, nome_produto, quantidade, preco_unitario, desconto, subtotal }]
  - Pagamentos: [{ forma_pagamento, valor, troco }]
  - Atualiza estoque (desconta quantidade vendida)
  - Atualiza pontos/total do cliente fidelidade

POST /api/vendas/:id/cancelar
  - Cancela venda e devolve estoque

--- FUNCIONALIDADES DO PDV (Frontend) ---

  - Busca por codigo de barras com auto-foco
  - Som de bip ao adicionar produto (Web Audio API 1200Hz)
  - Animacao flash ao adicionar item
  - Desconto por item individual (clique no valor)
  - Desconto global (valor ou porcentagem com preview)
  - Cliente fidelidade (busca, selecao, barra de info)
  - Calculo de troco em tempo real (verde = troco, vermelho = falta)
  - Multiplas formas de pagamento na mesma venda
  - Impressao de cupom profissional com dados completos
  - CPF na nota
  - Atalhos de teclado:
    F1 = Buscar produto
    F2 = Pagamento
    F4 = Cancela ultimo item
    F6 = Desconto geral
    F7 = Cliente fidelidade
    F8 = Abrir/fechar caixa
    F9 = Sangria
    F11 = Tela cheia (modo sem distracoes)
    ESC = Cancelar venda (ou fechar modal/busca)
    Ctrl+K = Busca global (funciona em todas as paginas)

  - Retorna: periodo, qtd_vendas, total_vendas, total_descontos, ticket_medio

  - Total custo e total venda de todo o estoque

  - Ranking de produtos por quantidade vendida
  - Params: data_inicio, data_fim, limit
  - Inclui: categoria, preco_medio

GET /api/relatorios/margem-lucro
  - Margem de lucro por produto
  - Retorna: receita, custo, lucro, margem_pct por produto

  - Classificacao ABC de produtos por faturamento

GET /api/relatorios/comparativo-mensal

GET /api/relatorios/vendas-por-hora

GET /api/relatorios/perdas-completo
  - Perdas por tipo (vencimento, avaria, furto) e por categoria
  - Retorna: resumo por tipo e detalhamento por categoria

GET /api/relatorios/fluxo-caixa
  - Fluxo de caixa com entradas (vendas) e saidas (sangrias + compras)
  - Params: data_inicio, data_fim
  - Retorna: entradas, saidas, saldo por dia

POST /api/relatorios/exportar
  - Endpoint para exportacao CSV server-side

--- EXPORTACAO (Frontend) ---

20. COMO INICIAR
================================================================================

  Desenvolvimento:  npm run dev     (com nodemon)
  Producao:         npm start       (node src/server.js)
  Desktop:          npm run desktop (abre no browser automatico)
  Windows:          Duplo-clique em "Iniciar Sistema.bat"
  Empacotar:        npm run empacotar (gera dist/SistemaGestao/)

  Login padrao:     admin / admin123
  URL:              http://localhost:3000
  Rede local:       http://[IP_DA_MAQUINA]:3000
